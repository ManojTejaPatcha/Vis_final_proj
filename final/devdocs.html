Under the Hood

PlanPilot runs three ML models entirely in the browser — no server, no data leaving the device. This page documents the architecture, model specifications, data schema, and file structure for contributors and researchers.

Stack: Vanilla JS · D3.js v7 · TopoJSON · No framework · Data: CMS Part D PUF 2023 · OECD Pharma Spending · 5 drugs × 8 insurers

ML Architecture

Three models. All in-browser. All client-side.

Model 1

Thompson Sampling Bandit

Tactic detection & confidence scoring · learning-engine.js

What it does

Maintains a Beta-distribution belief over how effective each of the 7 psychological tactics (anchoring, loss aversion, decoy…) has been on consumers similar to the current user. Samples to rank the tactics most likely being deployed in this pitch.

Implementation detail

Each arm (tactic × segment) has an independent Beta(α, β) prior. α increments on successful conversions for that tactic, β on resistant outcomes. Thompson sampling draws from each Beta distribution to rank tactics without an ε-greedy tradeoff.

Output

The "X% confidence" score on each tactic card in the Pitch Decoder. A 92% anchoring confidence means α/(α+β) ≈ 0.92 for the anchoring arm in this consumer's segment.

Model 2

k-Nearest Neighbors Cohort

Peer comparison & social proof verification · feature-engineering.js

What it does

Encodes the user profile (age, medications, state, budget sensitivity) as a feature vector and retrieves the k=5 most similar consumers from the interaction history using Euclidean distance in normalised feature space.

Implementation detail

Feature vector: [age/100, budgetOrdinal/2, medCount/5, regionOneHot×4]. Distance is L2-normalised. k=5 with no deduplication. Cohort summary computes conversion rate and top-ranked angles across neighbors.

Output

The cohort conversion rate in the narrative: "Of 5 similar consumers, 40% enrolled before seeing alternatives." Seeds: data/seed_interactions.json (100 synthetic interactions across 21 segments).

Model 3

Logistic Regression

Vulnerability scoring from profile features · ml-model.js

What it does

Trains on logged outcomes (converted / resisted) to learn which profile features predict susceptibility to sales tactics. Outputs a 0–100% vulnerability score using a sigmoid-activated weighted sum of the user's feature vector.

Implementation detail

Gradient descent, learning rate 0.01, 100 iterations. Weights initialised at zero. Features: age, budgetSensitivity (ordinal 1–3), medCount, valuesConvenience (binary), region (one-hot). Trained fresh from localStorage on each page load.

Output

The red/amber/green Vulnerability Score in the Pitch Decoder, plus the plain-English explanation below it ("Score elevated by: price-sensitive profile; 70+ age cohort"). Threshold: ≥70% = High, 40–70% = Moderate, <40% = Low.

Data Schema

insurer_pricing.json — one record per drug × insurer pair.

Pricing record (40 records total: 5 drugs × 8 insurers)

drug_id string Lowercase drug identifier: "eliquis", "trulicity", "ozempic", "jardiance", "xarelto"
drug_name string Display name: "Trulicity (Dulaglutide)"
insurer string Exact insurer name matching the dropdown. Must match exactly: "Kaiser Permanente", not "Kaiser".
patient_copay number Monthly copay in USD. Annual cost = patient_copay × 12. Used for verdict banner savings calculation.
prior_auth_required boolean If true, triggers the Hidden Requirements card "Prior Authorization Required" warning.
step_therapy boolean If true, triggers "Step Therapy Required" warning. Critical for GLP-1 drugs (Trulicity, Ozempic).
mail_order_available boolean Shown in comparison table. Also exploitable as a convenience upsell tactic.
network_size string "Nationwide", "Regional", or "Limited".
market_share number Percentage (0–100). Used in hero metrics and social proof tactic detection.
// Example: Trulicity × Kaiser Permanente { "drug_id": "trulicity", "drug_name": "Trulicity (Dulaglutide)", "insurer": "Kaiser Permanente", "patient_copay": 95, "prior_auth_required": true, "step_therapy": true, // triggers Hidden Req card "mail_order_available": true, "network_size": "Regional", "market_share": 5 }
